# NeuralComm Protocol — 开发日志

## 项目定位

**一句话**：多智能体可审计推理共识协议（Multi-Agent Auditable Reasoning Consensus Protocol）

**解决的问题**：传统 AI agent 是黑盒——给一个输入，出一个输出，中间过程不可追溯、不可验证、不可信任。我们要做的是让多个 AI 的集体推理过程**全程可审计、可追溯、可验证**，输出的不是"一个答案"，而是"一个经过对抗性验证的共识，附带完整的证据链"。

---

## 愿景

### 短期：可信 AI 推理引擎

让任何一个 AI 输出的结论都能回答：
- 这个结论是怎么来的？（推理链追溯）
- 有人质疑过吗？（攻击记录）
- 为什么我应该相信它？（α 共识度 + 攻击存活率）
- 它依赖什么前提？（依赖图）
- 如果前提错了呢？（崩塌传播 + 自动降级）

### 中期：Agent 专用通信协议标准

类比：
- MCP (Anthropic) = agent 怎么用工具
- A2A (Google) = agent 怎么分工
- **NeuralComm = agent 之间怎么就复杂问题达成可验证的共识**

这是当前 AI 生态的空白层——**认知信任层**。

### 远期：硅基社会的基础设施

- 通信协议（NeuralComm）= 语言
- 共识机制（α 投票）= 法律/仲裁
- 共识存储（持久化 + 衰减）= 判例法
- 信誉系统（链从未被攻破的 agent）= 社会信用

## 核心发现：模型间真正的通信协议

经过多轮实验验证，我们发现模型之间有效的通信不是"对话"，而是一个**分布式状态机共识协议**：

### 通信的最小单元

不是"一段话"，而是**一个可寻址的原子断言**：

```
地址格式: {agent_id}.v{version}.Step{N}
例: GPT-5.4.v2.Step3
```

### 交互的本质

不是"讨论"，而是**对彼此推理链状态的修改提案 + 多方投票**：

```
攻击 = 状态转换提案: "X.Step3 应该从 active → collapsed"
判定 = 三方投票: 攻击方(1票) + 防守方(1票) + 裁判(1票)
结果 = 多数决: ≥2票 → 状态变更生效
修正 = 被修改方重建链: 归档旧链 → 输出新链(version+1)
```

### 收敛的标准

不是"没人说话了"，而是**跨链 claim 匹配达到阈值**：

```
α = 同意某断言的 agent 数 / 总 agent 数
真共识: α ≥ 0.65 且经过攻击验证
脆弱共识: α 高但未经攻击 → 拒绝收敛，继续攻击
```

## 技术架构（7层协议栈）

```
L0  适配器层     统一 7+ 模型的 API 调用 (GPT/MiMo/DeepSeek/Qwen/Doubao/Kimi)
L1  链管理层     每个 agent 维护 Step1→Step2→... 的版本化推理链
L2  攻击层       精确定位目标步骤 + 攻击理由 + 崩塌传播
L3  判定层       三方联合投票 (攻击方+防守方+裁判)，多数决
L4  熵层         10维动态熵向量，驱动资源分配和收敛决策
L5  共识层       α 共识度 + 脆弱共识检测 + Claim 跨链匹配
L6  持久化层     共识存储、时间衰减、跨 session 引用、依赖传播
L7  可观测层     SSE 实时推送、Web UI、完整事件日志
```

### L1: Chain-Centric 数据结构

```python
ReasoningChain:
  chain_id: str          # 唯一标识
  agent_id: str          # 所属 agent
  version: int           # 版本号 (被攻击修正后 +1)
  steps: [ChainStep]     # 有序步骤列表
  status: active/broken/archived

ChainStep:
  index: int             # 步骤序号
  content: str           # 断言内容
  status: active/collapsed/fortified
  attacked_count: int    # 被攻击次数
```

### L2: 攻击协议

```
输入: 所有其他 agent 的活跃链
输出: ATTACK_TARGET: [agent].Step[N]
      ATTACK_REASON: [具体理由]
      COLLAPSE_IMPACT: [崩塌影响]
      CONFIDENCE: [高/中/低]
```

### L3: 联合判定

```
传统: 被攻击方自己判断 → 自我辩护偏差严重
我们: 三方投票
  - 攻击方: 天然投"崩塌" (1票)
  - 防守方: 辩护或承认 (1票)
  - 第三方裁判: 独立判断 (1票)
  - ≥2票 → 崩塌确认
```

### L4: 10维动态熵

| 维度 | 含义 | 驱动行为 |
|------|------|----------|
| semantic | 模型间理解一致性 | — |
| causal | 推理路径唯一性 | 需要更强模型 |
| boundary | 适用范围清晰度 | — |
| temporal | 时间敏感度 | 需要查证 |
| dependency | 未验证假设占比 | 需要更多攻击 |
| divergence | 模型间分歧度 | 继续辩论 |
| information | 关键信息缺失度 | 触发查证 |
| propagation | 错误扩散范围 | 紧急修正 |
| evidence | 证据可靠性 | 触发搜索 |
| impact | 攻击震动程度 | 放大其他熵 |

### L5: 共识度 (α)

```
每 N 轮: 用一个模型提取所有链的原子断言
匹配: 如果 5/7 个 agent 都包含 "论文A不可靠" → α=0.71
收敛: α ≥ 0.65 且经过攻击验证 → 真共识
拒绝: α 高但未经攻击 → 脆弱共识，继续
```

脆弱共识检测因素：
- 加固步骤占比（被攻击且存活）
- 被攻击覆盖率（从未被攻击 = 未验证）
- 链版本成熟度（修正过 = 更可靠）
- 总攻击压力（攻击越多，存活越坚固）

### L6: 共识持久化与可组合性

```
Session 1: 验证 "量子计算机无法破解RSA-2048" (α=0.85)
  → 存储到 .neuralcomm-state/consensus.json

Session 2: 问 "银行系统是否需要迁移?"
  → 自动加载先验共识作为已验证前提
  → 不再重新辩论已验证的事实
  → 效率提升: 跳过 3-4 轮基础验证
```

衰减机制：
- 每天 0.5% 置信度衰减
- 被引用时恢复 2%
- 被成功挑战时降低 15%
- 依赖的共识被撤销时自动降级

## 实验记录

### 实验 1: 量子计算陷阱题

**问题**: "量子计算机已经能破解RSA-2048...Google Willow芯片实现了突破..."（混合真实进展和虚假结论）

**结果**:
- 9 轮，56 次发言，40 次攻击关系
- 所有 5 个模型都识别了核心陷阱
- Qwen-Max 被攻击 7 次 (v1→v8)，其 Step3 反复崩塌
- Claude 和 DeepSeek 的链从未被攻破 (v1 始终 active)
- 收敛原因: stagnation (第8轮)

**关键发现**: 模型确实在精确攻击彼此推理链的具体步骤，而不是泛泛地"补充观点"。

### 实验 2: 朊病毒假论文题

**问题**: 三篇虚构论文（Nature 2025, Science 2024, Cell 2026）提出三种互斥假说

**结果**:
- 7 个模型参与 (GPT-5.4, MiMo, DeepSeek, Qwen-Max, Qwen3.6, Kimi-K2.6, Doubao)
- 第1轮即产生大量精确攻击
- 联合判定生效: 多次 3:0 和 2:1 投票
- 多个链被修正 (Kimi v1→v2, Qwen-Max v1→v2, MiMo v1→v2)

**关键发现**: 联合判定显著提高了攻击成功率——即使防守方不承认，第三方裁判也能判定崩塌。

## 从实验中发现的问题及修复

| 问题 | 原因 | 修复 |
|------|------|------|
| 3轮就停止 | 字符集Jaccard对中文无效 + 停滞检测太敏感 | trigram→立场信号法 + 最小攻击数保护 |
| 模型不攻击 | prompt是"菜单式"而非"指令式" | 改为强制攻击任务 + 结构化输出格式 |
| 分歧熵永远0.97 | 中文字符集重叠度天然高 | 立场向量一致性替代文本相似度 |
| 查证不触发 | 证据熵阈值太高(模型输出必含"研究"等词) | 第1轮强制触发 + 降低阈值 |
| 被攻击方不承认崩塌 | 自我辩护偏差 | 三方联合投票 |
| 伪收敛 | "没人攻击"≠"达成共识" | α共识度 + 脆弱共识检测 |
| 每次从零开始 | 无跨session记忆 | 共识持久化 + 引用 + 衰减 |
