# NeuralComm — 多模型神经链路协作理论

## 一句话

> 多个 LLM 不是平等投票，而是构成一条有反馈回路的神经链路——每个模型是一个神经元层，信号沿链路前馈传播，遇到错误时反馈回退，直到链路收敛。

---

## 核心问题

当前多模型协作的主流做法（投票、辩论、路由）都是平面的——模型之间没有层级、没有方向、没有反馈。这不是协作，是拼凑。

生物大脑的推理不是这样工作的。大脑有：
- **层级**：感觉 → 知觉 → 认知 → 决策
- **方向**：信号从低层向高层传播
- **反馈**：高层发现问题时，抑制信号回传低层，要求重新处理
- **稀疏性**：不是所有神经元都参与每次决策

---

## 核心机制

### 1. 信号，不是文本

模型之间传递的不是一段话，而是一个结构化信号：

```
{
  内容:     "建议终止进程1234"
  意图:     propose / inhibit / confirm
  置信度:   moderate
  推理轨迹: [步骤1→步骤2→步骤3]
  不确定性: ["不确定是否有子进程依赖"]
}
```

接收方精确知道：对方在说什么、有多确定、哪里不确定、希望我做什么。

### 2. 角色分层，不是平等讨论

每个模型被赋予一个神经元角色：

| 层 | 职责 | 适合的模型 |
|----|------|-----------|
| 感知层 | 快速反应，初步判断 | 快/便宜的模型 |
| 推理层 | 深度逻辑推演 | 通用能力强的模型 |
| 抑制层 | 找错，质疑，发出停止信号 | 不同厂商的模型（独立视角） |
| 元认知层 | 审视整条推理链的质量 | 最强的模型（稀疏激活） |

关键：**抑制层必须用和推理层不同的模型**。同一个模型不会质疑自己的盲点。

### 3. 前馈 + 反馈回路

```
感知 → 推理 → 抑制 → 元认知 → 输出
              ↑        |
              └────────┘
              (发现问题时回退)
```

- 正常路径：信号一路向前，每层处理后传给下一层
- 异常路径：抑制层发现逻辑漏洞 → 发出 INHIBIT 信号 → 推理层收到后重新推理
- 收敛条件：元认知层确认通过，或达到最大迭代次数

### 4. 稀疏激活

不是每次都调用所有模型。元认知层（最贵的模型）只在抑制层无法判断时才被激活。大多数简单任务在推理层就结束了。

### 5. 突触可塑性

每次执行后，链路自我调整：
- 快速收敛的路径 → 突触权重增强（下次更快）
- 反复回退的路径 → 突触权重减弱（下次可能跳过或换路）

链路本身在进化，不是静态的。

---

## 与现有范式的区别

| 范式 | 做法 | 问题 |
|------|------|------|
| 投票/集成 | 多模型做同一件事，取多数 | 浪费算力，无深度 |
| 辩论 | 两个模型互相反驳 | 可能永远不收敛 |
| 路由 | 根据任务选一个模型 | 没有协作，只是选择 |
| 流水线 | A的输出喂给B | 单向，无纠错能力 |
| **神经链路** | 分层前馈 + 反馈回退 | 有方向、有纠错、有收敛 |

---

## 关键洞察

1. **多模型的价值不在冗余，在结构** — 不同模型扮演不同认知角色，形成完整的思维链路
2. **回退比重试更高效** — 不是从头再来，而是精确回退到出错的那一层
3. **抑制和兴奋同样重要** — "停下来，这里有问题"和"继续，方向正确"是同等重要的信号
4. **置信度不能跨模型比较，但意图可以** — 用结构化信号替代原始概率
5. **最贵的模型应该最少被调用** — 稀疏激活，只在链路无法自行收敛时介入

---

## 实验验证

已通过 yunwu.ai 中转验证三模型链路：

```
GPT-5.4 (感知) → GPT-5.4 (推理) → Gemini 3.1 Pro (抑制) → Claude Opus 4.6 (元认知)
```

结果：
- 信号正确沿链路传播
- 每层输出被下一层接收并处理
- 抑制层独立验证推理层的逻辑
- 元认知层对整条链路做质量审查
- 链路在 4 轮迭代后收敛

## 一个类比

传统多模型协作 = 开会讨论（所有人同时说话，效率低）

神经链路协作 = 大脑思考（信号沿特定路径传播，遇到矛盾时回退修正，最终收敛为一个决策）
