# NeuralComm Protocol — 模型间神经通信协议

> 一种适用于所有 LLM 的底层通信算法，使多模型能以结构化方式传递认知状态、推理轨迹和协作信号。
> 不是让模型"投票"，而是让模型构成一个有反馈回路的神经链路。

## 核心理念

当前模型之间的通信只有"文本"——就像两个大脑之间只能通过写信交流。
NeuralComm 要做的是给模型之间建立"神经突触"——传递的不只是结论，而是完整的认知状态。

## 设计原则

1. 模型无关 — 任何 LLM (GPT/Claude/Gemini/本地模型) 都能接入
2. 不修改模型内部 — 纯外部协议层
3. 双向信号 — 支持前馈和反馈(回退)
4. 稀疏通信 — 只在必要时传递，不做冗余广播
5. 可进化 — 链路本身能根据历史表现自我优化

## 架构

```
输入信号
    |
    v
[感知神经元] ← 本地小模型(免费/快速)
    |
    v (前馈)
[推理神经元] ← 中等模型(便宜)
    |
    v (前馈)
[抑制神经元] ← 本地小模型(专注找错)
    |
    |--- 发现问题 → 反馈信号 → 回退到推理层 → 重新推理
    |
    v (通过)
[元认知神经元] ← 大模型(贵，稀疏激活)
    |
    v
最终输出
```

## 核心概念

### NeuralSignal (神经信号)
模型间通信的基本单元。不只是文本，而是包含:
- content: 内容
- intent: 意图 (propose/validate/challenge/inhibit/confirm...)
- certainty: 归一化置信度
- trace: 推理轨迹 (每一步的断言和依据)
- uncertainties: 显式标注的不确定性
- expectation: 对接收方的期望

### Neuron (神经元)
包装一个 LLM，赋予它角色:
- PERCEPTION: 感知层，快速反应
- REASONING: 推理层，深度思考
- INHIBITION: 抑制层，检测错误
- METACOGNITION: 元认知层，审视推理质量
- EXECUTION: 执行层，输出行动

### NeuralCircuit (神经回路)
将多个神经元连接成链路:
- 前馈传播: 信号沿链路向前
- 反馈回退: 抑制信号触发回退
- 稀疏激活: 只激活相关神经元
- 可塑性: 成功路径强化，失败路径弱化
